# 범위 계약

## 현재 범위 상태
- 현재 저장소는 일반-purpose 프론트엔드 스캐폴드 상태입니다.
- 실제 제품 도메인과 사용자 흐름은 아직 정의되지 않았습니다.

## 지금 확정된 범위
- 아키텍처 기반 유지
- 세션 복구 하네스 유지
- 정책-코드 동기화 하네스 유지
- GitHub Pages 자동 배포 유지

## 아직 확정되지 않은 범위
- 실제 사용자 문제
- 핵심 기능 집합
- 도메인 모델
- API 계약
- 데이터 저장 방식

## 범위 추가 규칙
- 새로운 범위는 먼저 `project-charter.md`에 반영합니다.
- 범위가 구조적 제약을 바꾸면 정책 문서도 같이 갱신합니다.
- 범위가 바뀌었는데 헌장이 비어 있으면 새 기능 구현보다 헌장 갱신을 먼저 합니다.
