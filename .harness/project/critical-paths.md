# Critical Paths

프로젝트에서 실수 비용이 큰 파일, 디렉터리, 외부 연동 지점을 선언합니다.

하네스는 이 문서의 `path` 값을 읽어 변경 파일과 비교합니다. 매칭되면 `harness:check`에서 더 강한 리뷰와 검증을 안내합니다.

## 작성 방법

| path | 왜 중요한가 | 권장 검증 |
| --- | --- | --- |
| `supabase/functions/**` | Supabase Edge Function은 서버 사이드 핵심 로직입니다. | `deno check` 또는 프로젝트 지정 검증 |
| `src/domain/**` | 도메인 규칙 변경이 사용자 동작에 직접 영향을 줍니다. | 관련 테스트, decision-log |
| `ios/**` | 네이티브 capability나 권한 설정이 앱 실행에 영향을 줍니다. | 로컬 빌드, 수동 확인 |

## 운영 원칙

- 구체적인 프로젝트 핵심 경로가 생기면 이 표에 추가합니다.
- 단순히 파일이 자주 바뀐다는 이유만으로 추가하지 않습니다. 장애, 데이터 손상, 배포 실패, 사용자 주요 흐름에 영향을 줄 때만 critical path로 둡니다.
- 경로가 매칭되면 변경 이유, 검증 결과, 수동 조치 여부를 `decision-log.md` 또는 업무 히스토리에 남깁니다.
