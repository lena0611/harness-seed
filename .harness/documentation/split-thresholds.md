# 문서 분리 기준

아래 중 하나라도 만족하면 문서를 분리하는 쪽을 우선 검토합니다.

## 분리 신호
- 한 문서가 서로 다른 목적의 섹션을 3개 이상 함께 담고 있을 때
- 세션 시작 시 필요한 내용과 참고용 상세 내용이 한 파일에 섞일 때
- 정책, 절차, 상태, 예외가 한 문서 안에서 함께 자랄 때
- 목차 없이 특정 정보를 찾기 어려워질 때
- 같은 문서에서 특정 하위 주제만 반복 수정될 때

## 권장 분리 방식
- `README.md` 또는 상위 문서: 인덱스 전용
- `*-guide.md`: 절차/운영 가이드
- `*-rules.md` 또는 `*-protocol.md`: 기준/규칙
- `*-context.md`, `*-memory.md`, `*-log.md`: 상태/이력
- `*.json`: 기계 판독용 레지스트리

## 분리 후 해야 할 일
1. 상위 인덱스 문서에 링크 추가
2. 세션 시작 순서에 영향을 주면 session harness 갱신
3. 관련 하네스의 registry 또는 설명 문서 갱신
