# 소비자 리뷰: 사용자 완료 승인 전 자동 최종화 금지

작성일: 2026-05-28

## 배경

하네스가 설치된 소비자 프로젝트에서 에이전트가 일반 작업 지시를 받은 뒤, 사용자의 추가 의견이나 완료 승인 여부를 확인하지 않은 상태로 검증, 커밋, 푸시까지 진행하는 문제가 관찰되었다.

## 문제

사용자의 `구현해줘`, `고쳐줘`, `문서 수정해줘` 같은 일반 작업 지시가 최종화 승인처럼 해석될 수 있다.

이 경우 사용자는 아직 결과물을 검토하거나 추가 의견을 줄 수 있는 상태인데도 아래 작업이 실행될 수 있다.

- `npm run build`
- `npm run test`
- `npm run harness:check`
- 배포
- git commit
- git push
- PR 생성

## 개선 원칙

- 작업 지시 직후 기본 상태는 `진행 중`이다.
- 사용자가 `완료`, `최종 검증`, `커밋`, `푸시`, `PR 생성`처럼 명시적으로 최종화 의사를 밝히기 전까지는 완료 승인 전으로 본다.
- 완료 승인 전에는 무거운 검증과 side effect 있는 작업을 실행하지 않는다.
- 필요한 검증은 실행하지 말고 `검증 후보`로 보고한다.
- git hook은 사용자가 commit/push를 승인한 뒤 실행되는 안전장치이며, 작업 완료 시점을 결정하는 장치가 아니다.
- Claude Stop hook은 기본적으로 `harness:check`를 자동 실행하지 않고, 명시적 승인 플래그가 있을 때만 실행한다.

## 수용 기준

- 에이전트가 일반 작업 지시만으로 build/test/harness:check/commit/push/PR 생성을 실행하지 않는다.
- commit/push 전에는 사용자 확인을 받는다.
- pre-commit/pre-push hook은 commit/push가 승인된 뒤에만 실행된다.
- Claude Stop hook은 기본적으로 `harness:check`를 자동 실행하지 않는다.
- 문서에는 검증이 완료 승인 뒤 최종화 단계에서 실행된다는 원칙이 명확히 기록된다.
