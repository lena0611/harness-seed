# /운영업무

운영 업무를 바로 구현하지 않고, 먼저 업무 유형과 JIRA 출처를 확정한 뒤 하네스 스킬을 선택합니다.

## 1. 대화창에서 먼저 물어볼 것

아래 값이 사용자 메시지에 이미 있으면 다시 묻지 않습니다. 없으면 작업 시작 전에 질문합니다.

1. 업무 유형을 선택하세요: `버그픽스`, `신규 기능`, `단순 리팩토링`, `기존 기능 개선`
2. 업무 요청 JIRA URL을 알려주세요.
3. JIRA 접근이 불가능하거나 제목/본문을 확인할 수 없으면 요청 개요를 한두 문장으로 알려주세요.

## 2. 스킬 선택

업무 유형에 따라 아래 하네스 스킬을 선택합니다.

| 업무 유형 | 기본 스킬 |
| --- | --- |
| `버그픽스` | `harness.bugfix-flow` |
| `신규 기능` | `harness.feature-flow` |
| `단순 리팩토링` | `harness.refactor-flow` |
| `기존 기능 개선` | `harness.enhancement-flow` |

선택한 업무 유형에 맞는 스킬이 `.harness/skills/registry.json`에 없으면 바로 구현하지 말고 다음을 수행합니다.

1. `.harness/skills/registry.json`에 프로젝트에 맞는 스킬 후보를 추가할지 판단합니다.
2. 공통화할 수 없는 프로젝트 전용 절차라면 `.harness/project/workflow-rules.md` 또는 `.harness/session/developer-input-queue.md`에 후보로 남깁니다.
3. 사용자에게 새 스킬로 진행해도 되는지 확인합니다.

## 3. 개발 진행

1. `CLAUDE.md`의 "항상 읽는 최소 기준"을 읽습니다.
2. `npm run harness:context -- "<업무 유형> <JIRA URL 또는 요청 개요>"`로 읽을 문서와 Selected Skills를 좁힙니다.
3. 선택된 업무 스킬의 `read`, `commands`, `outputs`, `records`를 기준으로 작업합니다.
4. 반복되는 도메인, 구조, 검증 규칙은 `.harness/project/*`로 승격할지 판단합니다.
5. 불확실한 질문은 `.harness/session/developer-input-queue.md`에 남깁니다.
6. 완료 전 `npm run harness:check`를 실행합니다. 실행하지 못하면 이유를 남깁니다.

## 4. 완료 후보와 업무요약 반자동 처리

에이전트가 구현과 검증을 마쳤더라도 바로 히스토리를 확정하지 않습니다. 먼저 사용자에게 완료 후보를 제시하고 승인 여부를 묻습니다.

```text
개발 완료 후보입니다.
- 요청 JIRA: <URL>
- 업무 유형: <유형>
- 핵심 변경: <요약>
- 검증: <실행 결과>

이 업무를 완료 처리하고 운영 업무 히스토리를 기록할까요?
```

사용자가 승인하면 `harness.work-summary` 흐름을 수행해 .harness/maintenance/work-history/연도별 폴더 아래에 이번 작업의 유지보수 히스토리를 남깁니다.

사용자가 아직 완료가 아니라고 하면 히스토리를 확정하지 않습니다. 남은 작업과 확인할 질문을 `.harness/session/next-session-reminder.md` 또는 `.harness/session/developer-input-queue.md`에 남깁니다.

`/업무요약` 명령은 수동으로도 사용할 수 있지만, 일반 사용자가 반드시 외워서 입력해야 하는 명령은 아닙니다. `/운영업무` 종료 시 사용자 승인 후 이어서 수행하는 반자동 절차입니다.

히스토리 작성 후에는 다음 승격 여부를 반드시 판단합니다.

- 중요한 구조 결정, 예외, 기준 충돌 해결이면 `.harness/session/decision-log.md`에도 기록
- 반복될 도메인/구조/검증 규칙이면 `.harness/project/domain-rules.md`, `architecture-rules.md`, `workflow-rules.md` 중 알맞은 곳으로 승격
- 다음 세션에서 이어야 할 항목이면 `.harness/session/next-session-reminder.md`에 기록
