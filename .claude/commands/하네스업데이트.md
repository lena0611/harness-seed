# /하네스업데이트

하네스 업데이트를 바로 실행하지 않고, 먼저 현재 설치 상태와 업데이트 대상을 분리해 확인합니다.

## 1. 먼저 후보 확인

```bash
npm run harness:outdated
```

출력에서 `baseHarness`와 `stackHarness`를 각각 확인합니다.

- `baseHarness`만 outdated이면 공통 하네스만 업데이트합니다.
- `stackHarness`가 outdated이면 스택 하네스 업데이트를 검토합니다.
- 둘 다 outdated이면 base와 stack의 update command를 확인하고, 필요한 순서를 사용자에게 설명합니다.

## 2. 업데이트 명령 선택

| 상황 | 명령 |
| --- | --- |
| 공통 하네스만 업데이트 | `npm run harness:update -- --base-only` |
| 스택 하네스 업데이트 | `npm run harness:update` |
| 공통만 재확인 | `npm run harness:outdated -- --base-only` |
| 스택만 재확인 | `npm run harness:outdated -- --stack-only` |

`npm run harness:update` 기본 동작은 스택 하네스 업데이트입니다. 공통 하네스만 올릴 때는 반드시 `--base-only`를 붙입니다.

## 3. 업데이트 후 확인

```bash
npm run stack:status
npm run harness:scan
npm run harness:check
```

확인할 항목:

- 공통 하네스 버전이 의도치 않게 낮아지지 않았는가?
- 스택 하네스가 요구하는 `baseHarness.minVersion`을 만족하는가?
- 프로젝트 소유 문서가 보존되었는가?
- `harness:check`가 통과하는가?

## 4. 기록

업데이트로 기준 해석이나 프로젝트 룰이 바뀌었으면 `.harness/session/decision-log.md`에 남깁니다.

업데이트 중 외부 권한, 토큰, GitLab 설정처럼 에이전트가 처리할 수 없는 항목이 있으면 `.harness/session/manual-actions.md`에 남깁니다.
